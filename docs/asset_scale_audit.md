# 画像アセット寸法一覧（調査）

`assets_loader.load_all_assets(PLAY_WIDTH, PLAY_HEIGHT)` ＝ **1280×760**。  
全面UI・スクロール背景とも原寸一致なら起動時スケールなし（`_prepare_play_screen_image`）。

**凡例**

| 状態 | 意味 |
|------|------|
| OK | ファイル原寸がコードの使用サイズと一致（起動時 scale 不要） |
| 要差し替え | ファイルを「あるべき原寸」で書き出し直すとよい |
| 欠損 | ファイルなし（コードはフォールバック） |

**スクロール**  
`draw_scroll` は画像幅でタイル（2枚 blit）。幅 **1280** ＝ プレイ幅と一致で継ぎ目なし。

## 11. 実行時のみスケール（素材寸法の目安）

コードで **毎フレーム** または **弾ごと** に `transform.scale` するもの。素材は基準 1 枚でよいが、キャッシュを増やす場合の目安。

| 用途 | 基準素材 | 実行時のサイズ | 備考 |
|------|----------|----------------|------|
| リップル弾 | `boss_ripple.png` 200×218 | 直径 8〜224 前後 | 半径に応じて都度 scale |
| 隕石破片 | `meteor.png` 80×48 | 約 34×20（42%） | 破片時のみ smoothscale |
| ボス5 突進 | `midboss5` 600×639 | ボス状態で動的縮小 | `boss5_update.py` |
| エンディング演出 | `ending.png` | 最大 1300×760 内で fit | 1 回 smoothscale |

## 12. 音声・フォント（参考）

画像ではないが配布容量に影響。

| 種別 | 代表ファイル | あるべき方針 |
|------|--------------|--------------|
| BGM | `*.ogg` | ゲーム内そのまま。再エンコードで MB 削減可 |
| SE | `*.wav` | 使用サイズそのもの。OGG 化で容量削減可 |
| フォント | `NotoSansJP-Regular.ttf` | 全文 5MB。サブセット化で **数百 KB〜1MB** 級に可能 |
